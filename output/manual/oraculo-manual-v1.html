Seção 0

O Que é o Oráculo

Para quem nunca usou inteligência artificial. Leia com calma — esses conceitos são a base de tudo que vem a seguir.

O problema que o Oráculo resolve

A Firma Abacaxi é uma produtora audiovisual de dois sócios. Lipe cuida da direção, da fotografia, da edição e do atendimento ao cliente, das propostas e da gestão de projetos. Jaya cuida do financeiro, administrativo e da operação. Em projetos intensos, ambos ficam sobrecarregados com tarefas operacionais que poderiam ser delegadas.

O Oráculo nasceu para absorver exatamente esse trabalho repetitivo e administrativo — para que Lipe e Jaya possam focar no que fazem de melhor: criar, dirigir e fotografar.

O que o Oráculo faz

  • Conduz briefings completos com novos clientes (faz as perguntas certas, na ordem certa)
  • Gera propostas comerciais e orçamentos detalhados em Word
  • Escreve roteiros, decupagens técnicas e planos de filmagem
  • Organiza a Ordem do Dia com prioridades claras (P0/P1/P2)
  • Cria calendários editoriais e posts para Instagram e LinkedIn
  • Pesquisa clientes, preços de mercado e referências na web
  • Acessa e atualiza os bancos de dados do Notion
  • Registra aprendizados de cada projeto para ficar cada vez melhor

O que o Oráculo NÃO faz

  • Não filma e não edita vídeos
  • Não substitui o olhar criativo de Lipe
  • Não toma decisões financeiras sem aprovação
  • Não altera nada no Notion sem pedir autorização primeiro
  • Não envia e-mails ou mensagens por conta própria

A analogia: o terceiro sócio digital

Pense no Oráculo como um colaborador muito bem treinado que leu todos os documentos da Firma — os preços, os clientes, o fluxo de trabalho, os aprendizados de cada projeto — e está disponível 24 horas por dia. Mas ao contrário de um estagiário que só anota, o Oráculo executa: pesquisa, calcula, escreve e organiza. Você orienta e aprova. Ele faz.

O que é Claude Code

Claude Code é um programa que você instala no computador e que traz uma inteligência artificial diretamente para dentro dos seus arquivos de trabalho. É diferente do ChatGPT que você usa no navegador por três razões fundamentais:

  1. Ele lê os seus arquivos. Quando você abre o projeto do Oráculo no VS Code, o Claude Code tem acesso a toda a documentação da Firma — o DNA, o fluxo de trabalho, as skills, o MEMORIA.md. Ele não precisa que você explique tudo do zero toda vez.
  2. Ele usa ferramentas. O Claude Code pode pesquisar na web, criar documentos Word, ler e escrever nos bancos de dados do Notion, criar arquivos — não só conversar.
  3. Ele é configurável. O arquivo CLAUDE.md na raiz do projeto é como o "manual de instruções" do Oráculo — define a personalidade dele, como identificar cada tipo de tarefa e qual módulo especializado (skill) usar.

O VS Code é o programa editor de código onde o Claude Code roda. Para Lipe e Jaya, o VS Code funciona apenas como a "janela" para conversar com o Oráculo.

Seção 1

Como Abrir e Usar

Abrindo o Oráculo

  1. Abra o VS Code
  2. Abra a pasta ORACULO - FIRMA ABACAXI (File → Open Folder)
  3. Abra o terminal integrado (Ctrl+J ou View → Terminal)
  4. Digite claude e pressione Enter
  5. O painel do Claude Code abre no lado direito — é aqui que você conversa com o Oráculo

Os 4 modos de operação

O Oráculo tem quatro modos. Você alterna com Shift+Tab ou clicando no indicador no rodapé do painel.

ModoO que fazQuando usar
PLANOráculo só lê e planeja. Não altera nada.Sempre primeiro em tarefas complexas ← recomendado
DEFAULTExecuta, mas pede confirmação em ações importantesUso geral do dia a dia
ACCEPT EDITSEdita arquivos sem pedir confirmação a cada umQuando você quer mais velocidade
AUTOFaz tudo automaticamente, sem pausasApenas quando você confia totalmente no plano

Regra de ouro: Para qualquer tarefa com mais de um passo (proposta, roteiro, ordem do dia), ative o PLAN MODE primeiro. O Oráculo apresenta o plano, você revisa, aprova, e só então ele executa.

Como escrever um bom pedido

Quanto mais contexto você dá, menos perguntas o Oráculo faz. A anatomia de um bom pedido:

[Contexto] + [O que quer] + [Formato esperado]

✅ BOM:
"Preciso de uma proposta para a CNV. Querem um documentário de 8 minutos
 sobre mediação de conflitos em Brasília. Prazo: 45 dias. Budget aberto.
 Pode gerar as 3 opções de valor."

❌ VAGO:
"Faz uma proposta"

@-mentions — referenciar arquivos diretamente

Em vez de deixar o Oráculo procurar arquivos (desperdiça processamento), você aponta direto. Digite @ seguido do nome do arquivo no chat:

@CLAUDE.md
@skills/proposta/SKILL.md
@MEMORIA.md
@docs/CONTEXTO_FIRMA.md

Exemplo prático:

"Usando @skills/proposta/SKILL.md como referência, gera uma proposta
 para o SuperHost. Briefing: vídeo institucional 2min, R$15k budget,
 2 locações em Brasília."

Comandos slash essenciais

Digite / no chat para ver todos. Os mais usados:

ComandoPara que serve
/planAtiva modo planejamento (equivale a Shift+Tab)
/clearLimpa o contexto — começa uma conversa nova, sem histórico
/compactComprime o histórico quando a sessão fica longa (economiza tokens)
/mcpVer integrações conectadas e status (Notion, Obsidian, etc.)
/doctorDiagnóstico completo do sistema
/costVer quanto custou a sessão atual

Use /clear entre tarefas diferentes. Contexto de uma proposta antiga misturado com uma nova vai gerar respostas confusas. Cada proposta, cada projeto — /clear antes de começar.

Revisando antes de aceitar

Quando o Oráculo edita um arquivo, o VS Code mostra as mudanças em verde (novo) e vermelho (removido) — igual ao sistema Git. Sempre revise o diff antes de clicar em Accept. Você nunca precisa aceitar às cegas.

Remote Control — pelo celular

No terminal do VS Code, digite:

claude --remote-control "Oráculo Firma Abacaxi"

Aparece um link. Abra o app Claude no celular → aba Code → sua sessão aparece. Do celular você pode mandar briefings por voz, ver o status de propostas e aprovar planos. O PC precisa ficar ligado.

Skill: Captação

✅ Ativa — Fase 1

O que faz: Conduz o processo completo de atendimento a um novo cliente — desde a qualificação até o momento em que a proposta está pronta para ser gerada.

Quando usar: Quando um novo cliente chega (Instagram, WhatsApp, indicação) e você quer conduzir um briefing estruturado.

Fluxo interno da skill:

  1. Qualificação inicial — classifica o lead como VERDE (avançar), AMARELO (cautela) ou VERMELHO (recusar)
  2. Briefing completo — 10 perguntas essenciais, uma por vez, sem sobrecarregar o cliente
  3. Pesquisa do cliente — busca web sobre a empresa, histórico, valores
  4. Registro no Notion — cria entradas em Clientes + CRM (com autorização)
  5. Perfil DISC — identifica o estilo de comunicação do cliente para adaptar a proposta
  6. Passa para a skill de Proposta

O que entrega: Briefing estruturado + cliente cadastrado no Notion + nota no Obsidian + proposta iniciada.

Exemplo de prompt de ativação
"Novo cliente chegou pelo Instagram. Nome: Ana Lima, empresa Vert. Quer um vídeo institucional. Vamos fazer o briefing."

Skill: Proposta

✅ Ativa — Fase 1

O que faz: Gera uma proposta comercial completa em arquivo Word (.docx), com orçamento detalhado, ancoragem de mercado e formatação profissional.

Quando usar: Quando o briefing já foi feito (pela skill de Captação ou direto por Lipe) e falta gerar o documento.

Fluxo interno da skill:

  1. Verifica se o briefing está completo — faz as perguntas que faltam
  2. Pesquisa de mercado — o que agências cobram por projetos similares
  3. Calcula o orçamento interno (não mostrado ao cliente): equipe + equipamento + locação + transporte + pós-produção + NF 7,28%
  4. Estrutura o documento com 7 seções
  5. Aplica ancoragem de mercado: "Agências cobram R$X–Y, nossa proposta: R$Z"
  6. Humaniza o texto (nunca soa como IA)
  7. Gera o arquivo Word em output/propostas/

Regras de ouro:

  • NF 7,28% sempre inclusa no total (nunca separada)
  • Margem mínima de 35% sobre o custo direto
  • Três opções de pacote apenas quando o budget do cliente é desconhecido
  • Pagamento: 50% na assinatura, 50% na entrega
  • Validade da proposta: 15 dias

O que entrega: output/propostas/NomeCliente_proposta_v1.docx

Exemplo de prompt de ativação
"Gera a proposta para o SuperHost com base no briefing que acabamos de fazer. Budget: R$15k. Pode gerar em uma opção só."

Skill: Pré-Produção

✅ Ativa — Fase 1

O que faz: Transforma um briefing aprovado em um plano de produção completo — roteiro, decupagem técnica cena por cena, plano de filmagem com horários e locações, e checklist de equipamentos.

Quando usar: Quando a proposta foi aprovada e é hora de planejar como filmar.

O que entrega:

  • Roteiro estruturado (Abertura / Cenas / Encerramento com CTA)
  • Decupagem técnica: tipo de plano, ângulo de câmera, movimento, iluminação, equipamento e duração por cena
  • Ordem de filmagem otimizada por locação e período do dia
  • Checklist de equipamentos
  • Arquivo em output/roteiros/NomeCliente_roteiro_v1.md
Exemplo de prompt de ativação
"Projeto Brasil Participativo aprovado. Documentário 8 minutos. Duas locações: UnB e Lago Sul. Vamos criar o roteiro e a decupagem técnica."

Skill: Produção

✅ Ativa — Fase 1

O que faz: Dá suporte durante e após a filmagem — checklists pré-set, diário de filmagem, registro de cenas gravadas e ocorrências.

Quando usar: No dia anterior à filmagem (preparação) e durante/após o set (registro).

Regra crítica: Máximo de 2 alterações no material sem custo adicional. A partir da 3ª alteração, gera um aditivo de contrato. O Oráculo ajuda a redigir e registrar isso.

Exemplo de prompt de ativação
"Vou filmar amanhã o projeto Cerrado Experience. Me dá o checklist de equipamentos e prepara o diário de set."

Skill: Gestão

✅ Ativa — Fase 1

O que faz: Organiza o dia, prioriza tarefas, controla prazos, acompanha o financeiro e gera comunicações operacionais.

Quando usar: No início de cada dia, em reuniões semanais, ou quando precisar de uma visão geral dos projetos.

Sistema de priorização:

  • P0 — urgente, faz hoje, não pode esperar
  • P1 — alta prioridade, faz hoje se possível
  • P2 — importante, mas pode ser esta semana

O que entrega: Ordem do Dia formatada por pessoa (Lipe/Jaya) + painel de projetos + controle financeiro (recebido / a receber / pipeline) + pautas de reunião + e-mails de follow-up.

Exemplo de prompt de ativação
"Qual é a ordem do dia? Temos 3 projetos ativos: Brasil Participativo, SuperHost e Evento UnB."

Skill: Conteúdo

✅ Ativa — Fase 1

O que faz: Cria calendários editoriais mensais, posts para Instagram e LinkedIn, e transforma projetos finalizados em múltiplas peças de conteúdo.

Os 5 pilares de conteúdo da Firma:

  1. Portfólio e resultados — mostrar o trabalho e o impacto
  2. Bastidores e processo — humanizar, mostrar o valor de cada detalhe
  3. Valores e identidade — projetos indígenas, arte, sustentabilidade
  4. Conhecimento técnico — construir autoridade, educar o cliente
  5. Empreendedorismo — a jornada de Lipe e Jaya construindo a Firma

Repropósito: De 1 projeto finalizado, a skill gera até 10 peças de conteúdo diferentes.

Exemplo de prompt de ativação
"Vamos criar o calendário editorial de junho. Projetos para mostrar: Brasil Participativo (documentário) e Cerrado Experience (making-of)."

Skill: Marketing Digital

✅ Ativa — Fase 1

O que faz: Estratégia de conteúdo, funil de conversão, tráfego pago e copywriting de alta conversão. Usa uma base de conhecimento construída a partir de apostilas de marketing salvas no Obsidian.

Quando usar: Quando precisar de decisões estratégicas sobre canais, campanhas ou quando o copywriting de uma proposta ou post precisa de mais impacto.

Exemplo de prompt de ativação
"Quero estruturar um funil de captação para novos clientes via Instagram. Qual é a estratégia recomendada para uma produtora do nosso porte?"

Skill: Humanizador

✅ Ativa — Fase 1 — SEMPRE usar em textos externos

O que faz: Reescreve qualquer texto para soar como Lipe ou Jaya escreveram — caloroso, direto, sem jargão corporativo, sem linguagem de IA.

Palavras e frases proibidas: "sinergia", "disruptivo", "solução inovadora", "Muito orgulhosos de apresentar", "Prezado cliente", "Venho por meio desta"

Regra absoluta: Todo texto que sai da Firma para um cliente externo (e-mail, proposta, post, mensagem) deve passar pelo Humanizador antes.

Exemplo de ativação
"Humanize este texto antes de eu enviar: [cola o texto]"

O Notion — Base de Dados Operacional

O Notion é o "escritório" do Oráculo — onde todos os projetos, clientes e tarefas são registrados. O Oráculo acessa o Notion via MCP (uma integração direta), o que significa que ele pode ler e atualizar os dados sem que você precise abrir o Notion manualmente.

Regra de segurança: O Oráculo sempre pede autorização explícita antes de criar, editar ou deletar qualquer registro no Notion. Ele nunca faz alterações por conta própria.

Fluxo ideal de uma proposta — passo a passo

1. Abra o VS Code na pasta do Oráculo

2. Terminal: claude
   → O painel do Claude Code abre

3. Ative o Plan Mode (Shift+Tab)

4. "Preciso de uma proposta para [cliente].
    [briefing que você tem]"

5. Oráculo planeja → você lê o plano → aprova

6. Oráculo executa:
   → Pesquisa o cliente na web
   → Lê skills/proposta/SKILL.md
   → Faz perguntas se faltar informação
   → Calcula orçamento interno
   → Escreve a proposta
   → Aplica o Humanizador
   → Gera Word em output/propostas/

7. Você revisa o arquivo → aceita

8. Oráculo pergunta: "Quer registrar no Notion?"
   → Você autoriza ou não

9. /clear → pronto para próxima tarefa

Boas práticas consolidadas

  • Plan Mode primeiro — antes de qualquer proposta, roteiro ou tarefa complexa
  • /clear entre tarefas — cada proposta é uma sessão nova
  • @-mentions — referencie os arquivos em vez de deixar o Oráculo procurar
  • Alimente o MEMORIA.md — ao fechar cada projeto, peça ao Oráculo para registrar o aprendizado
  • Humanizador sempre — todo texto externo passa pelo Humanizador antes de sair
  • Controle de versão — use _v1, _v2, _v3 nos nomes de arquivo para não perder histórico
  • /compact em sessões longas — quando a conversa passa de 1h, use /compact para economizar tokens

Erros comuns e como evitar

ErroComo evitar
Pedir proposta sem dar briefingSempre inclua: cliente, tipo de projeto, prazo, budget (se souber)
Dois projetos misturados na mesma sessão/clear entre um projeto e outro
Oráculo alterou o Notion sem quererSempre use Plan Mode antes de tarefas que envolvem o Notion
Texto soando como IA na propostaLembre de pedir o Humanizador ao final
NF cobrada separada do clienteNF 7,28% sempre já inclusa no total — nunca separada
Sessão lenta e respostas confusasUse /compact para comprimir o histórico

Glossário

Termos técnicos explicados em linguagem simples.

TermoO que significa na prática
Claude CodeO programa que você instala no PC e que roda a IA do Oráculo diretamente nos seus arquivos. Diferente do ChatGPT: lê seus documentos, usa ferramentas e é configurável.
VS CodeO editor de código onde o Claude Code roda. Para a Firma, é apenas a "janela" para conversar com o Oráculo.
MCPModel Context Protocol — a ponte que conecta o Oráculo a ferramentas externas (Notion, Obsidian, etc.). É o que permite o Oráculo ler e escrever no Notion sem que você precise abrir o navegador.
SkillUm módulo especializado que transforma o Oráculo em um especialista naquele assunto. Como contratar um especialista diferente dependendo da tarefa.
SubagenteUma versão simplificada e especializada do Oráculo que roda no browser (claude.ai). Útil no celular ou quando você não está no PC.
TokenA "moeda" de processamento da IA. Cada palavra lida ou gerada consome tokens. Por isso o /compact e o /clear ajudam a economizar — limpam tokens desnecessários.
Context windowA "memória de curto prazo" do Oráculo em uma sessão. Quando a janela fica cheia (sessão muito longa), o Oráculo começa a "esquecer" o início da conversa. Use /compact para gerenciar.
Plan ModeModo onde o Oráculo só lê e planeja — não altera nada. Você revisa o plano e aprova antes da execução.
@-mentionForma de referenciar um arquivo direto no chat do Oráculo. Em vez de ele procurar, você aponta: @CLAUDE.md, @skills/proposta/SKILL.md.
DiffA visualização das mudanças que o Oráculo quer fazer em um arquivo — verde = novo, vermelho = removido. Você aprova ou rejeita antes de aceitar.
VaultNome do Obsidian para uma pasta de arquivos. O "segundo cérebro" da Firma é o vault em cerebro/CEREBRO-ORACULO/.
PromptA mensagem que você escreve para o Oráculo. Um bom prompt tem: contexto + tarefa + formato esperado.
DecupagemO plano técnico de filmagem cena por cena — descreve ângulo de câmera, movimento, iluminação e equipamento para cada plano.
DISCModelo de perfil comportamental (Dominante, Influente, Estável, Cauteloso). O Oráculo identifica o perfil do cliente durante o briefing para adaptar o tom da proposta.
NF / Nota FiscalImposto de 7,28% sobre serviços audiovisuais. Na Firma, sempre já incluso no valor total cobrado ao cliente — nunca separado.
Ancoragem de preçoTécnica de apresentar um preço de referência maior (o que agências cobram) antes do preço real — faz o preço parecer mais razoável. O Oráculo usa automaticamente nas propostas.
